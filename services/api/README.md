# API backend (NestJS — à venir)

Ce dossier est **réservé** pour l’application HTTP NestJS (Clean Architecture, modules par domaine, DTO + `class-validator`).

L’app mobile consomme la même surface REST décrite côté client dans `src/shared/contracts/` et `src/lib/http/`.

Quand tu initialises Nest : `nest new .` depuis `services/api/` (ou déplace un projet généré ici) et expose une base URL utilisée par `EXPO_PUBLIC_API_BASE_URL`.
